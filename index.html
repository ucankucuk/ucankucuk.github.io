Java Collections(Java Koleksiyonları)

Java koleksiyonları : Java’da bir koleksiyon (collection - bazen container, ambar diye adlandırılır) nesnelerden olusan bir topluluğu bir arada tutan bir yapıdır.Yazılım geliştirirken bazen tek bir değişken içerisinde birden fazla eleman tutmak isteyebiliriz.Bunun gibi durumlarda Java'nın Collection sınıfını kullanırız.Bu sınıf : veri depolama ,depolanan veriyi çekme , veriyi işlemek ve depolanan veriler içerisinde veri aramak gibi işlemler için kullanılır.Bu sınıflar içerisinden depolanan veri sıralı veya sırasız olabilir.Ayrıca aynı öğe bazı koleksiyonlarda birden çok kez bulunabilir, bazılarında bulunamaz.

Java koleksiyonlarının avantajları olduğu gibi dezavantajları da vardır.

  • Avantajları:
    • Verileri tek bir değişkende toplamamızı ve bu toplanan verilerin işlenmesinde kolaylık sağlar.
    • Yazılımcı , koleksiyon sınıf içerisinde ki algoritmaları kullanarak yeni algoritma yazmak zorunda kalmaz.Bu sayede yazılım kolay anlaşılabilir ve tekrar kullanılabilir (software reuse) olmasını sağlar.
    • Yazılımcıya performans kaybı yaşatmadan hatta performansı arttıran kullanışlı algoritma ve veri yapıları sağlar.
    • Belli bir boyutları yoktur.İçerisine veri ekleyip çıkardıkça boyutları değişir.
    • Bu özelliği ile Arrays sınıfından öne geçer.Çünkü Java'da diziler önceden boyutlandırılır. Boyutundan fazla eleman eklememize olanak yoktur.
  • Dezavantajları :
    • Koleksiyonlara veri eklerken ; koleksiyonun veri tipine göre veri ekleyebiliriz.
    • Derleme anında veri tipi denetimi yapılmaz.

Collections Interface (Koleksiyon Arayüzü)

Collection arayüzü koleksiyon arabirimleri içerisinde bulunan bu arayüz hiyerarşik olarak en tepede bulunan birimdir.Java.util paketi içerisinde bulunan bu arayüz , bize birkaç arayüz ve bunlar altında bulunan sınıfları sunar.

  • Collection : Collection interface'inin hiyerarşik olarak en tepesinde bulunan , en genel sınıftır.
  • List : Liste yapısını örnek alır.İçerisinden bir elemandan birden fazla bulundurmaya izin verir.Elemanları sıralı biçimde tutar.
  • Set : İçinde saklanan elemanlar sırarız biçimde tutulur. İçerisinden bir elemandan birden fazla bulundurmaya izin vermez.
  • SortedSet : Verileri sıralı olarak tutar.İçerisinden bir elemandan birden fazla bulundurmaya izin vermez.
  • Queue : Kuyruk yapısını benimseyen koleksiyon tipidir. Liste türünden türetilmiştir.Ama indis değerleriyle işlem yapmaz.Bu öelliğiyle liste türünden ayrılır.
  • Map Elemanlarla bunlara karşılık verilen anahtar değerleriyle eşleştiren arayüzdür . Tekrar eden bir anahtar bulundurmaz. Grafikten de görüleceği gibi Map farklı yerde gösterilmiştir. Map arayüzü koleksiyon arayüzü ile benzerlik gösterir.Ancak Map arayüzünün elemanlarının her birini Unique bir anahtar değeriyle saklaması onu koleksiyonlardan ayırır.
  • SortedMap :Map arayüzünün özelliklerin taşır.Map'ten farkı anahtar değerleri artan bir sırada saklanır.

**Koleksiyon sınıfları ortak olarak birtakım metodlar kullanır.Metodlar bazı sınıflarda farklılık gösterebilir ama genel olarak aynıdır.

  • add() : Koleksiyon nesnesine referans verilen elemanı ekler.
  • remove() : Koleksiyon nesnesine referans verilen elemanı siler.
  • size() : Koleksiyon nesnesindeki eleman sayısını verir.
  • isEmpty() : Koleksiyon nesnesinin boş olup olmadığını kontrol eder.Boolean değer döner.
  • contains() : Koleksiyon nesnesi içerisinden referans olarak belirtilen elemanı arar.Booean değer döner.
  • iterator() : Koleksiyon nesnesi iterasyon nesnesine referans olarak göndermeye yarar.

Iterator

  • Iterator() metodu koleksiyon nesnelerini yinelememizi sağlar.
  • Iterator : koleksiyon nesnesi altındaki elemanlar arasında başlangıçtan sonuna kadar tek tek geçiş yapmamızı, elemanları güvenle silmemizi sağlar.
  • Iterator arayüzünün metodlar:
    • hasNext : Koleksiyon nesnesinin bir sonraki elemanının var olup olmadığını kontrol eder.Bir sonraki eleman varsa true yoksa false döner.
    • next : Koleksiyon nesnesinin bir sonraki elemana ulaşmasını sağlar.

SET(KÜMELER)

  • Collection interface'ini extends eder.(Miras alır).
  • Verileri sıralı bir şekilde tutmaz ayrıca bir elemandan sadece bir tane bulunur.
  • Set arayüzünü öğesi HashSet'tir.

HashSet

  • Set koleksiyonun bir örneğidir.Elemanların sıralı olması şartı yoktur.(Bu nedenle erişim sırası belirsizdir.)
  • Tekrarlı eleman içermez.
  • HashSet Kullanımına bir örnek:
package ucankucuk.github.io;

import java.util.HashSet;
import java.util.Iterator;

public class HashSetExample {

    public static void main(String[] args) {
        HashSet<String> hashSet = new HashSet<String>(); // HashSet koleksiyonu
                                                            // oluşturuldu.

        // HashSet nesnesine elemanlar eklendi.
        hashSet.add("Ali");
        hashSet.add("Mehmet");
        hashSet.add("Ahmet");
        hashSet.add("Veli");
        hashSet.add("Ali"); // Aynı eleman bir daha eklendi.
        hashSet.add(null);

        System.out.println(hashSet);

        // For döngüsüyle kullanım örneği
        for (String isim : hashSet) {
            System.out.println(isim);
        }

    }

}
  • Programın çıktısı :
    • [null, Ahmet, Veli, Mehmet, Ali]
    • null
    • Ahmet
    • Veli
    • Mehmet
    • Ali

  • Örnekte görüldüğü gibi aynı eleman hashset'e eklendiğinde önceki elemanı ezer ve hashset'te bir tane bulunur.
  • remove() metodu belirtilen eleman varsada yoksada **çalışır.Varsa siler yoksa false değeri döndürür.
  • Aynı elemanın tekrar eklemeye çalışılması erişim sırasını etkilemez.

LinkedHashSet

  • Elemanlar artan sırada sıralı değil giriş sırasına göre sıralıdır.
  • Tekrarlı eleman bulunmaz.
  • LinkedHashSet örneği :

LIST (Liste Yapıları)

  • Verileri bir dizi halinde depolar.
  • Dizilerde olduğu gibi burada da elemanların indis değerleri var.İndis değerleri ile elemanlara erişilebilir , arama yapılabilir.
  • Liste yapılarını dizilerden ayıran özellik liste yapılarının boyutlarının önceden ayarlanmış olmamasıdır.Bu nedenle daha kullanışlıdır.
  • Kopya eleman bulundurmaya izin verir.

ArrayList

  • Elemanları bir dizi biçiminde saklar ve boyut eleman eklendikçe büyür.
  • Elemanlara indeks değerleriyle erişmek ve silmek mümkün.
  • Not : dizilerden daha avntajlı olmalarına rağmen ; eğer depolanacak veri sayısı belli ve kolay kolay değişmiyorsa ArrayList kullanmak yerine Array kullanmak daha verimli olacaktır.
  • ArrayList sınıfı Array'den farklı olarak nesneleri depolayabilir.
  • ArrayList kullanımıyla ilgili örnek :
package ucankucuk.github.io;

import java.util.*;

public class ArrayListExample {

    public static void main(String args[]) {
        // ArrayList oluşturma.
        ArrayList<String> list = new ArrayList<String>();

        // ArrayList nesnesine eleman ekleme.
        list.add("Item1");
        list.add("Item2");
        list.add(2, "Item3"); // 2.indise Item3'ü ekledi
        list.add("Item4");

        System.out.println("ArrayList içeriği : " + list);

        /*
         * ArrayList nesnesi içerisinde arama yapma. Eğer aranan eleman varsa :
         * metod aranan elemanın indeksini döndürür. Eğer aranan eleman yoksa :
         * metod -1 değeri döndürür.
         */
        int pos = list.indexOf("Item2");
        System.out.println("Aranan elemanın indeksi : " + pos);

        /*
         * ArrayList nesnesinin boş olup olmadığının kontrolü Eğer liste boş
         * değilse false,boşsa true değerinin döndürür.
         */
        boolean check = list.isEmpty();
        System.out.println("Kontrol sonucu : " + check);

        // Listenin uzunluğu
        int size = list.size();
        System.out.println("Liste uzunluğu : " + size);

        /*
         * Listede eleman var mı kontrolü eğer eleman varsa true , yoksa false
         * döndür.
         */
        boolean element = list.contains("Item5");
        System.out.println("Arama sonucu : " + element);

        /*
         * İndeksi verilen elemanı getirme n elemanlı bir dizi de verilen indis
         * 0...(n-1) aralığında değilse; derleme sırasında
         * java.lang.IndexOutOfBoundsException hatası döner.
         */
        String item = list.get(0);
        System.out.println("Eleman : " + item);

        // Elemanları getirmek için aşağıda 3 yol gösterilmiştir.
        System.out.println("1.Yol : For döngü kullanarak");
        for (int i = 0; i < list.size(); i++) {
            System.out.println("Index: " + i + " - Item: " + list.get(i));
        }

        System.out.println("2.Yol : ForEach döngü kullanarak");
        for (String str : list) {
            System.out.println("Item is: " + str);
        }

        System.out.println("3.Yol : Iterator kullanarak");
        for (Iterator<String> it = list.iterator(); it.hasNext();) {
            System.out.println("Item is: " + it.next());
        }

        /*
         * Verilen indisdeki elemanı değiştirme. n elemanlı bir dizi de verilen
         * indis 0...(n-1) aralığında değilse; derleme sırasında
         * java.lang.IndexOutOfBoundsException hatası döner.
         */
        list.set(1, "NewItem");
        System.out.println("Değişimden sonra : " + list);

        /*
         * indeksi verilen elemanı silme n elemanlı bir dizi de verilen indis
         * 0...(n-1) aralığında değilse; derleme sırasında
         * java.lang.IndexOutOfBoundsException hatası döner.
         */
        list.remove(1);

        /*
         * Verilen elemanı silme. Eğer liste içeriğinde bulunmayan eleman
         * parametre olarak verilirse derleme sırasında hata alacaktır.
         */
        list.remove("Item3");

        System.out.println("Silme işlemlerinden sonra : " + list);

        // Array'den ArrayList'e çevirme.
        String[] simpleArray = list.toArray(new String[list.size()]);
        System.out.println("Array ArrayList'e çevirildikten sonra: " + Arrays.toString(simpleArray));
    }
}
  • Ekran çıktısı:
    • ArrayList içeriği : [Item1, Item2, Item3, Item4]
    • Aranan elemanın indeksi : 1
    • Kontrol sonucu : false
    • Liste uzunluğu : 4
    • Arama sonucu : false
    • Eleman : Item1
    • Retrieving items with loop using index and size list
    • Index: 0 - Item: Item1
    • Index: 1 - Item: Item2
    • Index: 2 - Item: Item3
    • Index: 3 - Item: Item4
    • Retrieving items using foreach loop
    • Item is: Item1
    • Item is: Item2
    • Item is: Item3
    • Item is: Item4
    • Retrieving items using iterator
    • Item is: Item1
    • Item is: Item2
    • Item is: Item3
    • Item is: Item4
    • Değişimden sonra : [Item1, NewItem, Item3, Item4]
    • The final contents of the arraylist are: [Item1, Item4]
    • The array created after the conversion of our arraylist is: [Item1, Item4]

Java StringBuilder & StringBuffer

  • StringBuilder nesneleri değiştirilebilir olması haricinde String nesnelerine benzer.
  • StringBuilder karakter dizisi içeren değişken uzunlukta ki diziler gibi davranır.
  • Herhangi bir noktada metod çağırımıyla uzunluk ve dizi içeriği değiştirilebilir.
  • Örneğin StringBuilder nesnelerinde append() , insert(), reverse() , replace() metodları vardır.
  • Çok sayıda dizileri bağlamada , StringBuilder nesnelerini kullanmak daha verimlidir.

Length & Capacity

  • StingBuilder sınıfı length() metoduna sahiptir.Bu özelliğiyle String sınıfına benzerdir.Ayrıca StringBuilder'ın capacity() metodu vardır.Bu metod StringBuilder'ın kapasitesini geri döndürür.Bu özellik String sınıfında yoktur.

package ucankucuk.github.io;

public class SetLengthMetodExample {
    public static void main(String[] args) {
        StringBuilder sb = new StringBuilder(); // 16 kapasiteli StringBuilder nesnesi oluşturuldu.
        System.out.println("Boş StringBuilder nesnesini kapasitesi = " + sb.capacity());
        System.out.println("Boş StringBuilder nesnesinin uzunluğu = " + sb.length());
        sb.append("0123456789ABCDEF");
        System.out.println("Eklenen = 0123456789ABCDEF" + " Eklemeden sonra uzunluk = " + sb.length());
        sb.append("UTKU");
        System.out.println("StringBuilder nesnesini içeriği = " + sb);
        System.out.println("Son eklemeden sonra uzunluk = " + sb.length());
        System.out.println("Son eklemeden sonra kapasite = " + sb.capacity());

        /*
         * 16 kapasitesindeki sb nesnesine 4 karakter daha eklediğimizde hata vermedi.
         * Çünkü StringBuilder otomatik olarak kapsiteyi arttırdı.
         */
    }

}
